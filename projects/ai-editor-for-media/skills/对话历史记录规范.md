# 对话历史记录规范

每次对话开始或结束时，将对话内容写入 `docs/chat-history/YYYY-MM-DD.md`。

## 规则

- **位置**：`docs/chat-history/` 目录
- **命名**：`YYYY-MM-DD.md`（按自然日）
- **追加**：如果当天文件已存在，追加内容而非覆盖
- **触发**：每次对话开始或结束时写入

## 为什么重要

保留所有开发对话的完整记录，便于追溯决策过程和项目演进。 ^[extracted]

## 应用场景

- 架构决策的讨论过程
- Bug 排查的思路演变
- 设计方案的权衡取舍
- 技术选型的理由记录
